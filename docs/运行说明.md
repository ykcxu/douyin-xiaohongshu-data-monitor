# 运行说明

## 1. 本地启动

### 安装依赖

```bash
copy .env.example .env
py -3 -m pip install -e .
```

### 执行数据库迁移

```bash
py -3 -m alembic upgrade head
```

### 启动 API

```bash
py -3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --app-dir src
```

### 启动初始化脚本

```bash
py -3 src/main.py
```

## 2. Docker 启动

```bash
cp .env.example .env
docker compose up --build
```

## 3. 当前关键环境变量

- `POSTGRES_HOST`
- `POSTGRES_PORT`
- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `REDIS_HOST`
- `REDIS_PORT`
- `BROWSER_STATE_DIR`
- `RAW_DATA_DIR`
- `SCHEDULER_ENABLED`
- `LIVE_STATUS_POLL_SECONDS`
- `DOUYIN_LIVE_PROVIDER`

## 4. 抖音 Provider 说明

当前支持两个 provider 值：

- `stub`
- `http`

说明：

- `stub`：本地开发占位实现，不访问真实抖音接口
- `http`：真实 HTTP 接入的占位骨架，已支持登录态文件、Cookie、请求上下文装配，但尚未完成真实接口逻辑

## 5. 登录态准备

建议流程：

1. 先通过 Playwright 扫码登录
2. 保存 `storage_state` 到 `BROWSER_STATE_DIR`
3. 通过系统已有逻辑写入 `browser_login_state`
4. 再使用 `DOUYIN_LIVE_PROVIDER=http` 进行后续联调

## 6. 当前建议的联调顺序

1. 创建监测账号
2. 创建抖音直播间
3. 确认数据库迁移完成
4. 先用 `stub` 跑通调度器
5. 再接入真实 `http` provider
6. 最后补小红书真实采集器

## 7. 样本联调建议

推荐先走一次样本联调：

1. 执行 `POST /seed/demo`
2. 使用 `data/samples/douyin_live_status_sample.json` 调 `POST /douyin/ingest/status-sample`
3. 再使用 `data/samples/douyin_live_comment_sample.json` 调 `POST /douyin/ingest/comment-sample`
4. 最后通过直播查询接口验证场次、快照、弹幕是否已入库
