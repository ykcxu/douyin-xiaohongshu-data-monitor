# 运行说明

## 1. Windows 本地启动

### 1.1 安装依赖

```bash
copy .env.example .env
py -3 -m pip install -e .
py -3 -m playwright install chromium
```

建议在 `.env` 里至少设置这几项：

```env
AUTO_CREATE_SCHEMA=true
DATABASE_URL=sqlite:///./runtime/app.db
SCHEDULER_ENABLED=false
DOUYIN_LIVE_PROVIDER=stub
APP_HOST=127.0.0.1
APP_PORT=8000
```

### 1.2 启动 API

```bash
py -3 -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --app-dir src
```

### 1.3 初始化联调数据

```bash
curl -X POST http://127.0.0.1:8000/seed/demo
```

### 1.4 扫码导出抖音登录态

方式一，直接运行 Python CLI：

```bash
py -3 -m app.cli.export_douyin_storage_state --account-id douyin_demo --operator admin
```

方式二，使用 PowerShell 包装脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\export_douyin_storage_state.ps1 -AccountId douyin_demo -Operator admin
```

执行后会自动打开 Chromium。你在浏览器里完成抖音扫码登录，然后回到终端按一次 Enter，系统会：

- 保存 `storage_state` 到 `BROWSER_STATE_DIR`
- 更新 `browser_login_state`
- 记录最新 `cookie_hash`

### 1.5 检查登录态

```bash
py -3 -m app.cli.inspect_login_state --account-id douyin_demo
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\inspect_douyin_login_state.ps1 -AccountId douyin_demo
```

### 1.6 切换到真实抖音联调模式

当登录态已经保存后，把 `.env` 调整为：

```env
DOUYIN_LIVE_PROVIDER=http
```

然后重启 API，即可开始接真实 provider 的本地联调。

## 2. 样本联调

推荐先跑一次样本链路，确认数据库、查询接口、JSONL 归档都正常：

1. `POST /seed/demo`
2. 用 `data/samples/douyin_live_status_sample.json` 调 `POST /douyin/ingest/status-sample`
3. 用 `data/samples/douyin_live_comment_sample.json` 调 `POST /douyin/ingest/comment-sample`
4. 用直播查询接口验证场次、快照、弹幕是否入库

## 3. Docker 启动

```bash
cp .env.example .env
docker compose up --build
```

## 4. 数据库迁移

开发或生产环境使用 Alembic：

```bash
py -3 -m alembic upgrade head
py -3 -m alembic revision -m "describe change"
```

如果是 Windows 本地 SQLite 联调，也可以通过 `AUTO_CREATE_SCHEMA=true` 自动建表。

- `POSTGRES_HOST`

- `POSTGRES_USER`

## 6. Douyin Provider 说明

当前支持两个 provider 值：

- `stub`
- `http`

说明：

- `stub`：本地开发占位实现，不访问真实抖音接口
- `http`：真实 HTTP 接入骨架，已经支持登录态文件、Cookie 和请求上下文装配，后续继续往里填真实请求与解析逻辑
